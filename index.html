Objetivos del Workshop

Primero Bienvenido a este Workshop de Flutter, el objetivo del mismo es brindarte las herramientas y orientación en tus primeros pasos utilizando Flutter.

La idea principal es que te lleves algo más que un tutorial paso a paso, un valor agregado a simplemente ver o seguir un tutorial de los miles que podrás encontrar.

Por ejemplo vamos a analizar los diferentes tipos de Widgets que ofrece Flutter uno de ellos será StatefulWidget, en cualquier lugar encontrarás mucha información acerca de su funcionamiento, incluso mucho más detallada, ahora, la idea de este Workshop es que además de conocer su funcionamiento, puedas saber que a la hora de re-utilizarlos es importante apoyarse en packages como

Flutter Hooks

Nota: Hooks are a new kind of object that manages a Widget life-cycles. They exist for one reason: increase the code-sharing between widgets and as a complete replacement for StatefulWidget.

Nuestro Proyecto - Feature - CA-001:

Es importante en principio y sin entrar en muchos detalles, crear una estructura básica para nuestro proyecto, en este punto Flutter no limita el uso de una arquitectura propia, pero puede resultar un gran desventaja si nuestro proyecto crece mucho y necesitamos mantener cierta independencia entre las diferentes capas como así también mantener nuestro código ordenado.

Para este Workshop usaremos la siguiente "capas" que nos ayudarán a entender y organizar más nuestro código.

presentation: La capa con la que va a interactuar el usuario, donde vivirá nuestra UI y los Widgets vinculados a la misma. En esta capa no deberá contener ninguna lógica (Algo que normalmente hacemos en nuestros primeros pasos, es normal ya que la lógica del negocio y el UI usan el mismo código).

application: La capa donde vivirán nuestros BLoC (Business Logic Components) que contendrán la lógica, es decir nuestros casos de uso, además de interactuar con la capa de presentación recibiendo y manejando las interacciones del usuario que despachen ciertos eventos.

infrastructure: La capa donde viven nuestros repositorios que se encargaran de comunicarse con el mundo exterior a través de una API o también obtener datos almacenados internamente por ejemplo si utilizamos Hive

config: Simplemente un directorio donde podremos almacenar configuraciones globales y el enrutamiento de la aplicación.

Nota: Esta estructura obviamente puede ser más compleja por ejemplo si agregamos una capa de dominio para independizar aún más las capas por ejemplo donde vivirán nuestras entidades, depende mucho de como quieran organizar su proyecto.

Pub Dev Packages

Nada nuevo... ahora si conocemos o ya estamos acostumbrados a utilizar ciertos packages podemos hacer uso de una extensión de vs code que te servirá de mucha ayuda sobretodo para llevar un control de tus dependencias y sus versiones actuales. Veamos como va esto.

pubspec.yaml: Si recordamos este archivo es el encargado entre otras cosas gestionar nuestras dependencias, podrán ver que podemos diferenciar entre dependencies y dev_dependencies, por si necesitamos incluir packages que solo usaremos en la fase de dev.

Vamos a utilizar Pubspec Assist para agregar una dependencia dev, que se llama lint (personalmente la considero fundamental incluirla al iniciar un proyecto) simplemente escribiremos el nombre del package o podemos consultar directamente pub.dev y seguir las intrucciones.

de cualquier manera tu resultado final debería ser el siguiente: 

...
dev_dependencies:
 flutter_test:
   sdk: flutter
 lint: ^1.1.1
...

Ahora solo resta obtener esta dependencia utilizando el siguiente comando:

flutter pub get

desde la consola en el directorio de nuestro proyecto donde se encuentra nuestro pubspec.yaml en el caso de vs code solo al detectar un cambio en este archivo intentará obtener las dependencias agregadas o simplemente apretando el siguiente botón en la barra superior..

Nota: Notaras que a su vez Pubspec Assist automáticamente hace tracking de las versiones de los packages incluidos en el archivo y informandote la última versión disponible y dándote la oportunidad que con solo un click puedas actualizarla :)

StatefulWidget:

StatefulWidget son utilizados si queremos conservar el estado de un widgets, básicamente en palabras simples es indicar que el widget cambio su estado y es necesario volverlo a construir para que se renderiza adecuadamente y el usuario vea el cambio en su pantalla. Por ejemplo si nuestro contador no fuese un StatefulWidget nuestra lógica se ejecutaría nuestra variable _counter se incrementa pero no verás ningún cambio en la interfaz, ya que nunca le informamos a Flutter que reconstruya ese elemento. Se entiende pero no queda claro? vamos a cubrir un poco el siguiente ejemplo así se entiende un poco mejor.

Nota: Estas explicaciones no son 100% técnicos con el fin de entender conceptualmente el funcionamiento, siempre es recomendable reforzar o guiarnos con la documentación oficial para cubrir todos los aspectos.

Para informar que este widget en particular cambio, debemos hacer uso del método propio de los StatefulWidgets setState() el cual se encargara de llamar al meotodo build() encargado de construir el widget. otro método importante que posee este tipo de widget es la posibilidad de tener un método initState() el cual servirá para iniciar el widgets o ejecutar alguna lógica antes de su primer renderizado: 

@override
void initState() {
   // TODO: implement initState
   super.initState();
 }

También mencionar el método dispose el cual se ejecutará cuando este StatefulWidget sea removido del Widget Tree. 

@override
void dispose() {
   // TODO: implement dispose
   super.initState();
 }

class _MyHomePageState extends State<MyHomePage> {
  int _counter = 0;

  void _incrementCounter() {

      _counter++;

Tenemos nuestra variable int _counter inicializada con el valor 0, la primera llamada al método build genera el siguiente widget.

genial, si presionamos el botón, este llama al método _incrementCounter() el cual incrementa nuestra variable _counter e informa a través de setState que el estado de ese widget cambio y tiene que ser reconstruido. Bien como hablamos al comienzo del Workshop, que tal si.... tocamos un poco esto usando Flutter Hooks y vemos la diferencia. Para esto vamos a hacer dos cosas, primero llevar el widget MyHomePage a presentation layer, aunque todavía no estaría del todo bien ya que tiene lógica incorporada en la UI, pero vamos de a poco.

Nota: Con fines de mantener la coherencia de la guía, vamos a preservar el código de main.dart y creamos un widget nuevo que extienda de HookWidget.

Obtendremos la misma funcionalidad con el siguiente código. 

import 'package:flutter/material.dart';
import 'package:flutter_hooks/flutter_hooks.dart';

class StatefulCounterHook extends HookWidget {
 final String title;

 StatefulCounterHook({Key key, this.title}) : super(key: key);
  @override
 Widget build(BuildContext context) {
   final ValueNotifier<int> _counter = useState(0);

   return Scaffold(
     appBar: AppBar(
       title: Text(title),
     ),
     body: Center(
       child: Column(
         mainAxisAlignment: MainAxisAlignment.center,
         children: <Widget>[
           Text(
             'You have pushed the button with Flutter Hooks this many times:',
           ),
           Text(
             '${_counter.value}',
             style: Theme.of(context).textTheme.display1,
           ),
         ],
       ),
     ),
     floatingActionButton: FloatingActionButton(
       onPressed: () => _counter.value = _counter.value+1,
       tooltip: 'Increment',
       child: Icon(Icons.add),
     ),
   );
 }
}

Si prestamos atención no existen muchos de los métodos mencionados como setState, dispose, initState , básicamente luce igual a un StatelessWidget, Flutter Hooks se encarga de gestionar el ciclo de vida del widget y al detectar un cambio en el valor de _counter informa que su estado cambió y ese widget debe reconstruirse. Buenisimo mas facil uso siempre Flutter Hooks! la idea de flutter hooks es mantener la reutilización de los métodos setState() initState() etc. Es recomendable utilizarlo cuando necesitemos realmente codigo reutilizable.

Por último existen muchos Hooks que ya viene listo para utilizar como por ejemplo el Hook de AnimationController ya lo veremos más adelante, lo importante es que sepas que puedes construir tus propios Hooks y adaptarlos a tus necesidades.
